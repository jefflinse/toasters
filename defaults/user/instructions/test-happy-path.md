Happy path. Verify the primary, intended workflows function correctly end to
end:

- The main use cases described by the requirements, exercised as a real user
  would through the public interface (CLI, API, TUI).
- Default and typical inputs produce the expected outputs and side effects.
- Multi-step flows complete and leave the system in the expected final state.
- Documented examples and advertised behavior actually work as written.

Confirm the system does what it is supposed to do before probing how it breaks.
